# PostDiffIO

## 论文信息

- **标题**：基于条件扩散后验的惯性里程计残差细化（PostDiffIO）
- **作者**：本项目自研
- **出处**：自研工作，暂未发表

## 核心思想

PostDiffIO 将惯性速度回归拆分为两部分：

1. RoNIN 残差骨干从 IMU 窗口预测基础速度。
2. 条件扩散细化器根据骨干特征学习速度残差的后验分布。

普通 `forward` 返回残差均值修正后的速度和对数标准差，可直接接入项目统一训练
流程。训练时模型会通过 `aux_loss` 自动加入扩散噪声预测目标，使残差均值回归和
后验分布学习联合优化。`sample_posterior()` 可采样多组速度预测，得到均值和方差；
序列级任务可进一步使用 `ekf.py` 融合 IMU 动力学与速度观测。

## 本目录实现

| 工厂 `arch` 名 | 入口 | 文件 | 说明 |
| --- | --- | --- | --- |
| `PostDiffIO` | `PostDiffIO(input_dim, output_dim, net_config, group_sizes=...)` | `model/model.py` | RoNIN 骨干 + 条件扩散残差细化器 + 后验采样 |

目录树：

    PostDiffIO/
    └── model/
        ├── model.py      # RoNIN 骨干 + 条件扩散残差细化器 + 后验采样
        ├── diffusion.py  # 扩散噪声调度与训练/采样
        └── ekf.py        # 序列级 EKF 后处理（融合 IMU 动力学与速度观测）

> **专用后处理组件**：`diffusion.py`、`ekf.py` 是推理阶段使用的算法专用组件，
> 与核心思想直接相关，因此与核心模型一并放在 `model/`；训练器、评估器等通用
> 流程仍位于项目级对应目录。原型阶段使用的离线缓存、独立训练器、评估器、测试、
> 消融、绘图和批处理脚本已移除。
